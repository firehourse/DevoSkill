# Thinking Phase Protocol

Use this protocol before writing or rewriting planning documents.

Its only job is to reduce wrong assumptions before planning artifacts are treated as executable.

## Load Order
1. Read `thinking-classification.md`
2. Read `thinking-reality-model.md`
3. Read `thinking-boundaries.md`
4. Read `thinking-delta.md`
5. Read `thinking-phasing.md`
6. Read `thinking-promotion.md`

Do not treat this file as the full rule body. It is the thin router for thinking-phase semantics.
