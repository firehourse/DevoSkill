# Thinking Reality Model Protocol

Use this protocol to gather only the minimum reality required to plan safely.

## Greenfield
- user goal
- success criteria
- expected system shape
- key constraints
- explicit non-goals

## Existing System
- current architecture
- current data/control flow
- movable vs immovable boundaries
- style/abstraction rules that must be preserved
- current pain or requested delta

## Hybrid
- inherited boundaries
- insertion points for the new capability
- new design vs inherited design
- migration risks and compatibility rules
